SPECIFICATION v0.1

Agent Transfer
Protocol

The missing standard for the agentic web

Every website is an API. Every API is discoverable. Every agent can act.

ATP is an open specification that lets any website expose its capabilities to AI agents through a simple, discoverable JSON manifest. No browser automation. No scraping. Just structured, authenticated, reliable access.

How It Works

Three Files.
Infinite Possibilities.

Step 01

Discover

/.well-known/agent.json

A JSON manifest at a well-known URL describes what the site can do, how to authenticate, and what capabilities are available. Just like robots.txt, but for agents.

Step 02

Authenticate

OAuth 2.1 / API Keys / Delegated

ATP defines standard auth flows. Agents present credentials; sites grant scoped access. Human delegation is built in from the start.

Step 03

Act

Typed API Calls

Agents call structured endpoints with full type safety. Every action is described semantically so any LLM can reason about what it does.

Specification

The Spec

A single agent.json manifest makes your entire site machine-readable. Here's what it looks like for an e-commerce store.

agent.json /.well-known/agent.json 
{
  "@context": "https://atp.dev/schema/v1",
  "@type": "AgentManifest",
  "name": "Acme Store",
  "description": "Online electronics retailer",
  "version": "1.0.0",

  "provider": {
    "name": "Acme Corp",
    "url": "https://acme.com",
    "contact": "api@acme.com"
  },

  "auth": {
    "schemes": [
      {
        "type": "oauth2",
        "flows": {
          "authorizationCode": {
            "authorizationUrl": "https://acme.com/oauth/authorize",
            "tokenUrl": "https://acme.com/oauth/token",
            "scopes": {
              "read:products": "Browse product catalog",
              "write:cart": "Manage shopping cart",
              "write:orders": "Place orders",
              "read:orders": "View order history"
            }
          }
        }
      },
      {
        "type": "apiKey",
        "in": "header",
        "name": "X-Agent-Key",
        "registration": "https://acme.com/developer/keys"
      }
    ],
    "agentIdentity": {
      "required": true,
      "format": "did:web"
    }
  },

  "rateLimit": {
    "requests": 1000,
    "window": "1h",
    "burstLimit": 50
  },

  "capabilities": [
    {
      "id": "search-products",
      "name": "Search Products",
      "description": "Search the product catalog by keyword, category, price range, or brand",
      "semanticType": "commerce:product-search",
      "endpoint": "/api/v1/products/search",
      "method": "GET",
      "parameters": [
        {
          "name": "q",
          "type": "string",
          "required": true,
          "description": "Search query"
        },
        {
          "name": "category",
          "type": "string",
          "enum": ["electronics", "clothing", "home"],
          "description": "Product category filter"
        },
        {
          "name": "price_min",
          "type": "number",
          "description": "Minimum price in USD"
        },
        {
          "name": "price_max",
          "type": "number",
          "description": "Maximum price in USD"
        }
      ],
      "response": {
        "type": "array",
        "items": {
          "$ref": "#/schemas/Product"
        }
      },
      "requiredScopes": ["read:products"]
    },
    {
      "id": "add-to-cart",
      "name": "Add to Cart",
      "description": "Add a product to the user's shopping cart",
      "semanticType": "commerce:cart-add",
      "endpoint": "/api/v1/cart/items",
      "method": "POST",
      "parameters": [
        {
          "name": "product_id",
          "type": "string",
          "required": true
        },
        {
          "name": "quantity",
          "type": "integer",
          "default": 1
        }
      ],
      "requiredScopes": ["write:cart"],
      "sideEffects": true,
      "confirmation": {
        "required": false
      }
    },
    {
      "id": "place-order",
      "name": "Place Order",
      "description": "Checkout and place an order for items in cart",
      "semanticType": "commerce:order-create",
      "endpoint": "/api/v1/orders",
      "method": "POST",
      "parameters": [
        {
          "name": "shipping_address_id",
          "type": "string",
          "required": true
        },
        {
          "name": "payment_method_id",
          "type": "string",
          "required": true
        }
      ],
      "requiredScopes": ["write:orders"],
      "sideEffects": true,
      "confirmation": {
        "required": true,
        "message": "This will charge the user's payment method and create a binding order."
      }
    }
  ],

  "workflows": [
    {
      "id": "purchase-flow",
      "name": "Product Purchase",
      "description": "Complete flow from search to purchase",
      "steps": ["search-products", "add-to-cart", "place-order"]
    }
  ],

  "schemas": {
    "Product": {
      "type": "object",
      "properties": {
        "id": { "type": "string" },
        "name": { "type": "string" },
        "price": { "type": "number" },
        "currency": { "type": "string" },
        "description": { "type": "string" },
        "category": { "type": "string" },
        "in_stock": { "type": "boolean" },
        "image_url": { "type": "string", "format": "uri" }
      }
    }
  },

  "policies": {
    "training": "deny",
    "inference": "allow",
    "caching": {
      "allowed": true,
      "maxAge": 3600
    },
    "attribution": "required"
  }
}
M

Manifest Metadata

Schema context, provider info, and versioning. Agents know exactly what they're talking to.

A

Authentication

Multiple auth schemes — OAuth 2.1, API keys, and DID-based agent identity. Flexible yet standardized.

C

Capabilities

Each action is a typed endpoint with semantic types like commerce:product-search. Agents understand intent, not just structure.

W

Workflows

Multi-step sequences with defined ordering. Agents can execute complex flows without guessing the next step.

P

Policies

Explicit training/inference permissions, caching rules, and attribution requirements. Site owners stay in control.

S

Side Effects & Confirmation

Dangerous actions are flagged with sideEffects: true and optional confirmation messages. No silent charges.

Philosophy

Design Principles

01

Simplicity First

One JSON file. No new runtime. No build tools. If a developer can write robots.txt, they can write agent.json.

02

Discovery by Convention

/.well-known/agent.json is the single entry point. Like robots.txt for capabilities. No out-of-band configuration.

03

Semantic, Not Just Structural

Capabilities have semantic types (commerce:product-search) so agents understand intent, not just parameters.

04

Auth is First-Class

OAuth 2.1, API keys, and delegated agent identity (DID:web) are built into the spec. No guessing how to authenticate.

05

Human-in-the-Loop by Default

Side effects require explicit declaration. Irreversible actions require confirmation. Agents can't silently charge credit cards.

06

Context-Window Aware

The manifest is designed for LLM consumption. Progressive disclosure: summary first, details on demand. Fits in context windows.

Tooling

atp-convert

Turn any website into an Agent API. One command generates a draft manifest from your existing site.

01

Analyzes the website and identifies forms, actions, and data structures automatically.

02

Infers capabilities from page semantics — login forms become auth flows, search bars become search capabilities.

03

Generates a draft agent.json manifest ready for review and customization.

04

Optionally generates API route handlers that wrap the existing website's functionality.
